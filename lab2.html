Programming lab 2: Extracting Data from Wikipedia

TA: Andrés Ortega

Introduction

Various Wikipedia articles count with a small table at the top right corner of the screen named infobox. The infobox cointains short data that may be shared with other articles such as birth date in the case of people or population in the case of countries.
Your task for this programming lab is to extract this data and convert it into a CSV file.

Assignment (80%)

Download all pages from the category 20th-century mathematicians from the export pages tool and write a program that does the following:

  • (70%) Read the downloaded XML file and output a CSV file named infoboxData.csv with the format: 
    <name>,<birth date>,<death place>,<birth place>,<death place>,<alma mater>,<fields>
    using the information available from the pages infoboxes. If there is no information available for a given field (e.g. there is no infobox) the corresponding field should be left blank. You might find Data.CSV and Text.Regex.TDFA useful.
  • (10%)
    • Your lab will be tested using the -Wall, -Wmissing-local-signatures and -Wmissing-signatures options.
    • Annotate (in English) your source code using Haddock. Documentation should explain your solution for the assignment.
    • Improve your implementation with HLint.

Before submitting your code, which includes your README, clean it up! Clean code:

  • Does not have long lines (at most 80 columns).
  • Has an uniformly indentation (we recommended two characters).
  • Has a consistent layout.
  • Has type signatures for everything (included where-clauses and let-definitions).
  • Has good comments.
  • Has no junk (unused code, commented code, unnecessary code).

  • Has no unnecessary spaces at the end of a line, or empty lines at the end of a file. (See: Fixing trailing whitespace issues with Emacs ).
